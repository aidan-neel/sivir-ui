  in CI until 003's step 1 lands. 001 can proceed alone, but don't tag a
  release before at least 003's CI step is in.
- **004 before 001's first real publish (soft)**: the CLI default registry
  domain ships inside every consumer's `sivir.json`; get the canonical domain
  decided (004 step 1) before the package is live.
- 002, 005, 006, 007 are fully independent of each other and of the rest.
- **008, 009, and 010 are immediate correctness blockers** and can run in
  parallel on non-overlapping areas; rerun their combined browser suite before
  merging.
- **011 after 009 and 010**: Command composition removes Command-only theme
  tokens, and the canonical theme contract defines which variables are public.
- **012 after 009**: use the typed context pattern established by the
  Command/Modal composition across the remaining compound components.
- **013 last**: it turns the repaired behavior and artifact into release gates.
  Do not tag until 003, 005, 008, 009, 010, 011, 012, and 013 satisfy their done
  criteria.
- Plans 001-007 assume the 2026-07-06 baseline. The 2026-07-14 follow-up
  baseline was package tests 15/15, docs unit/SSR 463 passing, and browser 134
  passing with 3 ColorPicker interactions skipped. A green baseline is not a

## Findings considered and rejected (so nobody re-audits them)

- **".env committed to the public repo"** (SEC-01): false — `git ls-files` and
  `git log --all` show `apps/registry/.env` was never tracked; root
  `.gitignore` covers it. (It does hold live credentials on disk — rotation is
  cheap insurance; noted in plan 002's maintenance notes.)
- **hono CORS/path-traversal advisories affect the registry** (SEC-04/08/09):
  mis-attributed — hono enters the tree only via `@prisma/dev` (Prisma's dev
  toolchain); the registry serves via Elysia. Tracked as accepted residual in
  plan 005.
- **CLI path traversal as a live vulnerability** (SEC-05): the file list comes
  from the registry snapshot bundled inside the npm package, not from remote
  data; `theme install` writes a fixed filename. Kept as defense-in-depth
  (plan 003 step 4), rejected as a vulnerability.
- **Turbo cache keyed per-SHA is an anti-pattern** (PERF-03): rejected — key
  on `github.sha` + `restore-keys` prefix fallback is the canonical
  actions/cache + turbo pattern and gives warm caches across commits.
- **List virtualization** (PERF-05): speculative for typical list sizes;
  revisit with real numbers if consumers hit 1,000+ items.
- **CI registry tests need DATABASE_URL** (DX-07): rejected — the tests mock
  `@lib/prisma` in-memory (hermetic by design) and CI is green.
- **Installation docs mislead about the install model** (DOCS-03): rejected —
  the page correctly leads with the CLI flow; this was an opinion, not a defect.
- **Studio has no publish/export flow** (DIR-03): false — `publishTheme()` in
  the studio posts to `/api/themes`, which proxies to the registry.
- **?raw docs migration incomplete** (DIR-04): stale — 42 of 43 component
  route dirs already use `?raw` examples.
- **Community themes gallery missing** (DIR-05): false —
  `apps/docs/src/routes/themes/+page.svelte` exists.
- **`state_referenced_locally` warnings in tabs/code-block/collapsible**: the
  prop-seeding is deliberate (comments explain SSR-correct first paint) with
  `$effect` syncs directly below; warnings are benign. Silencing them is
  cosmetic and was not planned.
- **Delete `docs/superpowers/` + `docs/theme-studio/` planning artifacts**
  (DEBT-07): kept — they are decision records that grounded this audit
  (several subagent false-positives were rejected because of them).
- **`raw.css` at repo root** (DEBT-08): orphan utility, one file, zero
  imports; not worth a plan — fold into any passing cleanup.
